### Attacks & Vulnerabilities

[### First AI-powered ransomware PoC spotted (2 minute read)](https://www.theregister.com/2025/08/26/first_aipowered_ransomware_spotted_by/?utm_source=tldrinfosec)

PromptLock, the world's first known AI-powered ransomware, uses OpenAI's gpt-oss-20b model to generate malicious scripts. It is currently only a proof-of-concept and has not been seen in active attacks. PromptLock demonstrates how AI can simplify cybercriminal operations.

[### NX Compromised to Steal Wallets and Credentials (4 minute read)](https://semgrep.dev/blog/2025/security-alert-nx-compromised-to-steal-wallets-and-credentials/?utm_source=tldrinfosec)

The popular NX build system was compromised with malicious versions published to npm that steal cryptocurrency wallets, API keys, and credentials from at least 1,400 affected users. The malware creates repositories prefixed with "s1ngularity-repository" in victims' GitHub accounts. It uniquely leverages the Claude Code CLI or Gemini CLI to execute file system searches for sensitive data through AI prompts, making detection more challenging for traditional security tools. Users should immediately check for unauthorized repositories, update to version 21.4.1, and rotate all potentially compromised credentials, including GitHub tokens, npm keys, and SSH keys.

[### Data I/O Becomes Latest Ransomware Attack Victim (2 minute read)](https://www.darkreading.com/cyberattacks-data-breaches/data-io-ransomware-attack?utm_source=tldrinfosec)

Data I/O, a tech manufacturer, suffered a ransomware attack on August 16 that led to major outages. The incident's impact remains unclear as the investigation is still underway, but restoration of operations has begun. Costs may be substantial.

### Strategies & Tactics

[### The One Where We Just Steal The Vulnerabilities (CrushFTP CVE-2025-54309) (7 minute read)](https://labs.watchtowr.com/the-one-where-we-just-steal-the-vulnerabilities-crushftp-cve-2025-54309/?utm_source=tldrinfosec)

A flaw (CVE-2025-54309) in CrushFTP servers allows attackers to gain administrative access if certain features are disabled. Researchers used a honeypot sensor network to catch hackers exploiting this race condition in real time and confirmed that two coordinated HTTP requests could add a rogue admin account. The vulnerability impacts thousands of vulnerable servers and is being exploited in the wild, highlighting the need for organisations to patch and monitor for unusual activity.

[### Beware the False False-Positive: How to Distinguish HTTP Pipelining From Request Smuggling (6 minute read)](https://portswigger.net/research/how-to-distinguish-http-pipelining-from-request-smuggling?utm_source=tldrinfosec)

Sometimes, when people are hunting for HTTP request smuggling vulnerabilities, they actually run into false positives caused by connection reuse. Some scenarios where connection reuse may be necessary for finding vulnerabilities include situations where the frontend server only reuses the upstream connection if the client connection was reused, connection-state attacks, and client-side desync attacks. James Kettle created a Custom Action for Burp Suite called `Smuggling or pipelining?` to distinguish between a genuine HTTP smuggling attack and an HTTP pipelining false-positive.

[### Behind the Coinbase breach: Bribery emerges as enterprise threat (8 minute read)](https://www.csoonline.com/article/4042522/behind-the-coinbase-breach-bribery-is-an-emerging-enterprise-threat.html?utm_source=tldrinfosec)

Threat actors like The Com and Scattered Spider bribed TaskUS workers in India with up to $2,500 to steal data from 70,000 Coinbase customers, resulting in a $20 million ransom demand that Coinbase refused, offering the same as a bounty. The breach revealed a shift from phishing to insider targeting through bribery, primarily targeting customer support staff with data access. Coinbase fired 226 workers, opened a US support hub, and enhanced controls. Experts advise red-team exercises to test employee responses to bribery attempts worldwide.

### Miscellaneous

[### New Sni5Gect Attack Crashes Phones and Downgrades 5G to 4G without Rogue Base Station (2 minute read)](https://thehackernews.com/2025/08/new-sni5gect-attack-crashes-phones-and.html?utm_source=tldrinfosec)

Sni5Gect is a framework that can downgrade 5G connections to vulnerable 4G, crash phone modems, and bypass authentication by passively sniffing unencrypted messages between base stations and phones during the initial connection phase before authentication. The attack achieves 80% sniffing accuracy and 70-90% message injection success rates from up to 20 meters away, exploiting the window before NAS security context establishment without requiring a rogue base station. Tests successfully targeted phones from OnePlus, Samsung, Google, and Huawei. The GSMA has acknowledged the vulnerability as CVD-2024-0096.

[### Anthropic's auto-clicking AI Chrome extension raises browser-hijacking concerns (5 minute read)](https://arstechnica.com/information-technology/2025/08/new-ai-browser-agents-create-risks-if-sites-hijack-them-with-hidden-instructions/?utm_source=tldrinfosec)

Anthropic's new Claude for Chrome AI extension, now in limited preview, allows AI to control browser actions but poses significant security risks. Researchers found these AI agents are vulnerable to hidden, malicious website instructions, with attacks succeeding up to 23.6% of the time. Although new safety measures cut success rates, experts warn browser-integrated AI may be fundamentally unsafe for now.

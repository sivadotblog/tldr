[### Making File Encryption Fast and Secure for Teams With Advanced Key Management (5 minute read)](https://dropbox.tech/security/file-encryption-teams-advanced-key-management?utm_source=tldrinfosec)

Dropbox describes how it solved for Hardware Security Module (HSM)-backed, team-based file encryption, where files are encrypted at rest and access can be removed instantly while also prioritizing usability at scale. End-to-end encryption would make it impossible for Dropbox to offer file content search, and traditional team-based key management, where a top-level key is used to derive unique keys for each file, would be prohibitively slow due to Dropbox's file storage architecture. To solve this problem, Dropbox introduced namespace encryption keys, which are derived from an HSM-backed top-level key and are used to derive team encryption keys. When sharing files, Dropbox can temporarily share the namespace encryption key while it reencrypts the blocks with team encryption keys.

It found 35+ affected tags, and the risk persists via inherited bases and CI pipelines. Recommendations: avoid those tags, rebuild from patched bases, and scan for the issue in your images.

### Miscellaneous

[### MuddyWater's DarkBit Ransomware Cracked (2 minute read)](https://www.bleepingcomputer.com/news/security/muddywaters-darkbit-ransomware-cracked-for-free-data-recovery/?utm_source=tldrinfosec)

Cybersecurity firm Profero has cracked the Iranian-backed MuddyWater's DarkBit ransomware, allowing clients to recover their data without paying a ransom. Profero found that the key generation method that DarkBit uses is weak and can be brute-forced. It also discovered that VMDK files on ESXi servers are sparse, and much of the content that its clients needed could be recovered without needing to brute-force the decryption. Profero is not publicly releasing its decryptor but urges victims to contact them.

[### How Tool Calling Works in LLMs (4 minute read)](https://newsletter.scalablethread.com/p/how-tool-calling-works-in-llms?utm_source=tldrinfosec)

LLMs use tools, external functions, or APIs when a user makes a request that the model cannot fulfill given its internal knowledge base. During inference, an LLM is provided with a series of tool definitions as part of its input prompt or context window, much like providing a human with a series of external functions and their documentation. When the LLM detects that a tool must be called, it ranks potential tools based on their relevance to the task and then calls the appropriate tool to address the user's query.

### Attacks & Vulnerabilities

[### Brosix and Chatox Promised To Keep Your Chats Secured. They Didn't (3 minute read)](https://databreaches.net/2025/08/05/exclusive-brosix-and-chatox-promised-to-keep-your-chats-secured-they-didnt/?utm_source=tldrinfosec)

Chatox and Brosix are both communications platforms for personal and team use owned by Stefan Chekanov. A security researcher discovered an unencrypted backup of chat logs containing 155.3GB of unique, compressed files containing over 65M stored messages and 73K files for Chatox and over 37M messages and 191K files for Brosix. Upon notification, the Brosix backup was secured, but the Chatox backup is still unsecured.

[### WinRAR Zero-Day Exploited to Plant Malware on Archive Extraction (2 minute read)](https://www.bleepingcomputer.com/news/security/winrar-zero-day-flaw-exploited-by-romcom-hackers-in-phishing-attacks/?utm_source=tldrinfosec)

Researchers from ESET discovered a new WinRAR vulnerability that is being actively exploited in phishing campaigns. Unpatched versions of WinRAR and Windows versions of UnRAR, RAR, and UnRAR.dll can be tricked into using a path specified by a maliciously crafted archive file as the extraction directory. This vulnerability is being exploited by the Russian hacking group RomCom to install malware.

[### Linux-Based Lenovo Webcams' Flaw Can Be Remotely Exploited for BadUSB Attacks (4 minute read)](https://thehackernews.com/2025/08/linux-based-lenovo-webcams-flaw-can-be.html?m=1&utm_source=tldrinfosec)

Security researchers discovered vulnerabilities in Lenovo 510 FHD and Performance FHD webcams that allow attackers to remotely transform them into BadUSB devices capable of injecting keystrokes and installing malware. The Linux-based webcams lack firmware validation, enabling complete compromise and persistent re-infection even after system wipes and OS reinstalls. Lenovo has released firmware version 4.8.0 to address the vulnerabilities following responsible disclosure in April.

### Strategies & Tactics

[### Repeater Strike: Manual Testing, Amplified (5 minute read)](https://portswigger.net/research/repeater-strike-manual-testing-amplified?utm_source=tldrinfosec)

Gareth Heyes of PortSwigger writes about his experience writing a Burp Suite AI extension to automate hunting for IDOR and similar vulnerabilities. The extension, Repeater Strike, scans requests and responses to generate regular expressions that can be used to identify vulnerabilities in your traffic. While the extension worked in small tests, the AI struggled with longer responses, creating regular expressions that aren't site specific, and detecting more complex vulnerabilities.

[### Why XSS Persists in This Frameworks Era? (24 minute read)](https://flatt.tech/research/posts/why-xss-persists-in-this-frameworks-era/?utm_source=tldrinfosec)

Modern frameworks block most HTML XSS through automatic escaping, but XSS persists when developers bypass protections with WYSIWYG editors, markdown parsers, or flawed sanitizers. Frameworks don't guard against JavaScript XSS via URL attributes (href/src with javascript:) or eval in OAuth flows, redirects, or postMessage. Even trusted libraries may cause XSS if misused, and defenses like CSP and WAF are often bypassed, so developers must treat all input as untrusted and reliably use protections.

[### I SPy: Escalating to Entra ID's Global Admin With a First-Party App (10 minute read)](https://securitylabs.datadoghq.com/articles/i-spy-escalating-to-entra-id-global-admin/?utm_source=tldrinfosec)

Researchers from DataDog discovered an attack vector where an attacker could abuse an Entra ID service principal (SP) with the Cloud Application Administrator or Application Administrator roles or the `Application.ReadWrite.All` permission to obtain global admin. The attacker uses the compromised SP to hijack the Office 365 Exchange Online SP to add a new federated domain, which they can then abuse to log in as any hybrid user. The researchers reported this vulnerability to Microsoft, but it was dismissed as a configuration issue.

[### 0day.today Archive (GitHub Repo)](https://github.com/vulncheck-oss/0day.today.archive?utm_source=tldrinfosec)

VulnCheck-OSS created a GitHub repo archiving tens of thousands of exploits from 0day.today, a public exploit database that went offline in early 2025 and returned without its decade of data. ### Miscellaneous

[### API Keys for Bedrock: A Brief Security Overview (5 minute read)](https://medium.com/@adan.alvarez/api-keys-for-bedrock-a-brief-security-overview-2133ed9a2b3f?utm_source=tldrinfosec)

AWS has introduced the ability to create Bedrock API keys, which are special API keys that are explicitly scoped to Bedrock. Long-term API keys are tied to an automatically generated or existing IAM user, can be configured with an expiration of up to 100 years, and are composed of a magic 3-byte prefix, a human-readable section, and a base64-encoded secret. Short-term API keys are scoped to a specific user's session and have the format `bedrock-api-key-[BASE64]` where [BASE64] is a base64-encoded pre-signed URL.

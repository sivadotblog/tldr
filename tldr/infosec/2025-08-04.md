[### Attackers Exploit Link-Wrapping Services to Steal Microsoft 365 Logins (2 minute read)](https://www.bleepingcomputer.com/news/security/attackers-exploit-link-wrapping-services-to-steal-microsoft-365-logins/?utm_source=tldrinfosec)

Cloudflare's Email Security team observed a new campaign in which threat actors abuse link-wrapping services from Proofpoint and Intermedia to phish users for Microsoft 365 credentials. Attackers abused Proofpoint link-wrapping in various ways, including multi-tiered redirect abuse with URL shorteners via compromised accounts. In the Intermedia campaign, threat actors sent a message impersonating a ‚ÄúZix‚Äù secure message notification or a Teams notification that would redirect users to a fake login page to harvest Teams credentials.

[### CL-STA-0969 Installs Covert Malware in Telecom Networks During 10-Month Espionage Campaign (5 minute read)](https://thehackernews.com/2025/08/cl-sta-0969-installs-covert-malware-in.html?m=1&utm_source=tldrinfosec)

Chinese state-sponsored threat actor CL-STA-0969 conducted a 10-month espionage campaign that targeted Southeast Asian telecom networks. The campaign used custom malware, including AuthDoor, GTPDOOR, and ChronosRAT, to enable remote control and location tracking. The group deployed advanced evasion techniques, including DNS tunneling and traffic routing through compromised mobile operators, while maintaining high operational security. There was no evidence of data exfiltration despite the attackers installing multiple backdoors and scanning tools across critical telecommunications infrastructure.

[### Critical Alert: Akira Ransomware Surge Targets SonicWall Firewalls (3 minute read)](https://www.criticalpathsecurity.com/critical-alert-akira-ransomware-surge-targets-sonicwall-firewalls/?utm_source=tldrinfosec)

Akira ransomware attacks targeting SonicWall SSL VPN connections have surged since July. Exploiting CVE-2024-40766 (CVSS 9.3) and unpatched SonicOS devices with disabled MFA, attackers rapidly escalate from VPN access to system encryption within hours, affecting hundreds of thousands of devices across multiple SonicWall generations. Organizations must immediately patch to the latest firmware, enable MFA on all VPN accounts, and restrict firewall management access to trusted IP ranges.

### Strategies & Tactics

[### Kubernetes Security Fundamentals: Authorization (7 minute read)](https://securitylabs.datadoghq.com/articles/kubernetes-security-fundamentals-part-4/?utm_source=tldrinfosec)

Kubernetes handles authorization by utilizing multiple authorization providers in the cluster requested in the order specified to the kube-server as flags on startup. Each provider can explicitly approve or deny the request or respond with ‚Äúno opinion.‚Äù If all providers respond ‚Äúno opinion,‚Äù the request is denied. Providers include AlwaysAllow and AlwaysDeny, which allow or deny every request; Node Authorizer, which restricts the access of the kubelet user; Attribute-Based Access Controls (ABAC), which enables cluster-admins to define authorization rules based on the request's characteristics statically; Role-Based Access Controls (RBAC), which is the most commonly used provider and maps permissions to roles; and Webhook, which is widely used in cloud provider clusters like AKS, GKE, and EKS.

[### Exploiting Trust: How Signed Kernel Drivers Fuel Modern Kernel Level Attacks on Windows (11 minute read)](https://www.group-ib.com/blog/kernel-driver-threats/?utm_source=tldrinfosec)

An emerging pattern in kernel level attacks is to use a signed kernel loader to load a signed second stage kernel driver or reflectively load an unsigned kernel driver. These signed drivers are often signed via the Windows Hardware Compatibility Program (WHCP). The WHCP requires an extended validation (EV) certificate, which is designed to increase the security of the program. However, the EV certificate application process requires little human interaction, and there are dark web sellers of EV code-signing certificates.

[### Building Tamper-Proof Media Apps: C2PA vs CPP Deep Dive for Developers (14 minute read)](https://dev.to/veritaschain/building-tamper-proof-media-apps-c2pa-vs-cpp-deep-dive-for-developers-52g4?utm_source=tldrinfosec)

This technical deep dive compares C2PA (the industry standard backed by Adobe and Sony) with the emerging Content Provenance Protocol (CPP) for building tamper-proof media applications that prove content authenticity. CPP's key differentiator is its "Completeness Invariant," which uses XOR hash sums to mathematically detect deleted evidence‚Äîa critical capability for legal proceedings, insurance claims, and regulatory compliance that C2PA cannot provide architecturally. The article provides Swift and Python implementations of cryptographic patterns, including Secure Enclave signing, Merkle tree construction, RFC 3161 timestamp anchoring, and privacy-preserving human attestation without storing biometric data.

[### Kubernetes v1.35: A Better Way to Pass Service Account Tokens to CSI Drivers (4 minute read)](https://kubernetes.io/blog/2026/01/07/kubernetes-v1-35-csi-sa-tokens-secrets-field-beta/?utm_source=tldrinfosec)

Kubernetes v1.35 added the ability for CSI drivers to opt-in to receiving service account tokens via the `secret` field in `NodePublishVolumeRequest`. Prior to this update, service account tokens were received via the `volume\_context` field, which could lead to secrets being included in gRPC request logs if drivers did not implement custom sanitizer logic. To opt in to this feature, CSI drivers should implement the `serviceAccountTokenInSecrets' field as well as implement logic for backwards compatibility.

[### The Dual-Use Dilemma of AI: Malicious LLMs (13 minute read)](https://unit42.paloaltonetworks.com/dilemma-of-ai-malicious-llms/?utm_source=tldrinfosec)

KawaiiGPT is an open-source malicious LLM. This article analyzes the efficacy of these models at producing ransomware notes, data exfiltration scripts, and lateral movement detection.

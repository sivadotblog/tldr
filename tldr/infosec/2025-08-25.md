[### Booking.com phishing campaign uses sneaky '„Çì' character to trick you (1 minute read)](https://www.bleepingcomputer.com/news/security/bookingcom-phishing-campaign-uses-sneaky-character-to-trick-you/?utm_source=tldrinfosec)

Attackers are using the Japanese hiragana character '„Çì' (n) in phishing domains to impersonate Booking.com, creating URLs that appear legitimate but redirect victims to credential-stealing sites. The campaign exploits how the character renders similarly to the Latin 'n' in many fonts, making malicious domains harder to detect. [### The Silent, Fileless Threat of VShell (8 minute read)](https://www.trellix.com/blogs/research/the-silent-fileless-threat-of-vshell/?utm_source=tldrinfosec)

Attackers embed base64-encoded Bash commands in RAR filenames that run automatically during directory listing with unsafe patterns like eval "echo $f" or for f in \*. The attack downloads loaders specific to architecture that fetch XOR-encrypted VShell backdoor payloads, decrypts them in memory, and executes as fake kernel threads such as [kworker/0:2] to evade detection. This method bypasses defenses because antivirus doesn't scan filenames, the malware doesn't touch disk after decryption, and execution happens via common shell operations without needing executable permissions.

[### Introducing Unit 42's Attribution Framework (16 minute read)](https://unit42.paloaltonetworks.com/unit-42-attribution-framework/?utm_source=tldrinfosec)

Unit 42's three-tier attribution framework tracks threats from activity clusters to named threat actors, requiring rigorous evidence like Diamond Model mapping and Admiralty System scoring for source reliability (A-F) and credibility (1-6). [### rre-burp (GitHub Repo)](https://github.com/jumpycastle/rre-burp?utm_source=tldrinfosec)

The RRE Burp extension automates detecting Recursive Request Exploits by tracing API call chains from protected resources to find unauthenticated upstream APIs exploitable to bypass access controls. It uses Shannon entropy analysis (default threshold 3.0) to identify valuable backend tokens and parameters, then traces requests to find low-trust inputs that produce these high-value outputs. Demonstrated at DEF CON 33, it scales to find vulnerabilities across streaming services ("Streamageddon"), revealing how API chain flaws can bypass paywalls.

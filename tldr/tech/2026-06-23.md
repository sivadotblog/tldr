[### The Optimal Amount of Slop is Non-zero (19 minute read)](https://www.slater.dev/2026/06/the-optimal-amount-of-slop-is-non-zero/?utm_source=tldrnewsletter)

Software requires different levels of verification depending on its use case. Casual software has limited distribution and loose quality constraints. Business software needs to work, or organizations suffer losses. Acute software is mission-critical and merits the highest levels of scrutiny. The amount of permissible AI-slop decreases as the risks of the software failing increase.

[### You should use AI for reviewing code especially when the diff is huge (2 minute read)](https://simianwords.bearblog.dev/you-should-use-ai-for-reviewing-code-especially-when-the-diff-is-huge/?utm_source=tldrnewsletter)

A human code reviewer's role is now to transfer knowledge to the AI that it doesn't know. This workflow is best in places where each line of code isn't sacred.

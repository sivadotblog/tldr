[### Compound Engineering: How Every Codes With Agents (13 minute read)](https://every.to/chain-of-thought/compound-engineering-how-every-codes-with-agents?utm_source=tldrnewsletter)

In traditional engineering, each feature makes the next feature harder to build because of new edge cases, interdependencies, and issues. In compound engineering, each feature makes the next feature easier to build because compound engineering creates a learning loop for agents and team members so that each bug, failed test, or insight gets documented and used by future agents. AI's knowledge of the codebase grows along with it, making future development work faster. This post provides a high-level sense of how to start compound engineering with AI.

[### An SVG is all you need (4 minute read)](https://jon.recoil.org/blog/2025/12/an-svg-is-all-you-need.html?utm_source=tldrnewsletter)

SVGs are vector graphics in a simple XML format. They are supported on almost every device and platform. With embeddable scripts to make them interactive, SVGs are way more capable than many people realize. It is entirely feasible to implement an entire data analysis pipeline in an SVG. A completely self-contained SVG file can fetch data from a versioned repository, process that data, and generate visualizations without any server-side magic required.

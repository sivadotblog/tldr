### News & Trends

[### Cloudflare targets 2029 for full post-quantum security (9 minute read)](https://blog.cloudflare.com/post-quantum-roadmap/?utm_source=tldrdevops)

Cloudflare accelerated its post-quantum cryptography timeline to 2029 after Google revealed a breakthrough algorithm that dramatically speeds up breaking elliptic curve cryptography and Oratomic demonstrated that neutral atom quantum computers could crack P-256 encryption with just 10,000 qubits. The company is now prioritizing quantum-resistant authentication over encryption-only protections, warning that Q-Day—when quantum computers can break current cryptography—could arrive as early as 2029-2030, much sooner than the previously expected 2035+ timeline.

[### Assessing Claude Mythos Preview's cybersecurity capabilities (46 minute read)](https://red.anthropic.com/2026/mythos-preview/?utm_source=tldrdevops)

Claude Mythos Preview demonstrates a major leap in cybersecurity capabilities, autonomously finding and exploiting zero-day vulnerabilities across major systems, chaining multiple bugs into sophisticated exploits, and even enabling non-experts to generate working attacks. These capabilities emerge from general improvements in reasoning and autonomy, creating short-term risks where attackers may benefit first, but likely long-term advantages for defenders who can use the same tools to secure software at scale.

[### Stop Answering the Same Question Twice: Interval-Aware Caching for Druid at Netflix Scale (9 minute read)](https://netflixtechblog.com/stop-answering-the-same-question-twice-interval-aware-caching-for-druid-at-netflix-scale-22fadc9b840e?source=rss----2615bd06b42e---4&amp;utm_source=tldrdevops)

Netflix built an experimental caching layer for its Apache Druid database that now handles over 10 trillion rows and ingests up to 15 million events per second, reducing query load by 33% and improving response times by 66%. The system splits time-series results into independently cached buckets with age-based TTLs (5 seconds for fresh data, up to 1 hour for older data), allowing dashboards to serve 84% of result data from cache while only querying Druid for the newest data—a critical improvement during high-profile live events when dozens of engineers monitor the same dashboards simultaneously.

HCP Terraform introduces IP allow lists to restrict access to organization resources and agent pools based on trusted CIDR ranges, improving security and compliance. [### S3 Files and the changing face of S3 (20 minute read)](https://www.allthingsdistributed.com/2026/04/s3-files-and-the-changing-face-of-s3.html?utm_source=tldrdevops)

S3 Files allows S3 data to be accessed as a filesystem by layering a file interface (via EFS) on top of object storage, using a “stage and commit” model that preserves the fundamental differences between file and object semantics instead of forcing them into a single system. This approach reduces data friction by letting data stay in one place while supporting both file-based and object-based access patterns without compromise.

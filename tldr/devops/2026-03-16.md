### News & Trends

[### Serverless Workspaces in Azure Databricks is now Generally Available (2 minute read)](https://www.databricks.com/blog/serverless-workspaces-azure-databricks-now-generally-available?utm_source=tldrdevops)

Databricks launched Serverless Workspaces in general availability on Azure, allowing teams to deploy fully managed workspaces in minutes without designing networks or provisioning compute infrastructure. The service integrates with Microsoft's Azure ecosystem and Unity Catalog for governance, offering a faster alternative to Classic Workspaces for teams prioritizing speed over custom networking configurations.

[### Twenty years of Amazon S3 and building what's next (5 minute read)](https://aws.amazon.com/blogs/aws/twenty-years-of-amazon-s3-and-building-whats-next/?utm_source=tldrdevops)

Amazon S3 celebrated its 20th anniversary, having grown from storing 1 petabyte across 400 nodes at launch in 2006 to now holding over 500 trillion objects and handling 200 million requests per second across hundreds of exabytes of data. Despite this massive scale, AWS has reduced S3 pricing by about 85% (from 15 cents to just over 2 cents per gigabyte) while maintaining complete backward compatibility with code written two decades ago.

[### Claude Code with Docker: Local Models, MCP, Sandboxes (3 minute read)](https://www.docker.com/blog/run-claude-code-with-docker/?utm_source=tldrdevops)

Claude Code can now be run with Docker to give developers full control over their AI coding assistant through local model execution via Docker Model Runner, connections to 300+ pre-built MCP (Model Context Protocol) servers for tool integration, and isolated sandboxes that let the AI agent autonomously modify files and install packages without risking the host machine. The setup uses Docker's Anthropic-compatible API and one-click MCP server deployment to create a secure, private development workflow that works consistently across Mac, Windows, and Linux.

### Opinions & Tutorials

[### Building AI Teams: How Docker Sandboxes and Docker Agent Transform Development (5 minute read)](https://www.docker.com/blog/building-ai-teams-docker-sandboxes-agent/?utm_source=tldrdevops)

Docker Agent is an open source framework for building collaborative AI agent teams with specialized roles that coordinate tasks like design, coding, testing, and debugging. Running them inside Docker Sandboxes provides microVM isolation so agents can execute commands and modify files autonomously without risking the host system.

[### Codegen Is Not Productivity (13 minute read)](https://www.antifound.com/posts/codegen-is-not-productivity/?utm_source=tldrdevops)

Lines of code are a poor measure of programming productivity because programming is primarily about understanding problems and managing complexity rather than producing large amounts of code. Although LLMs can accelerate writing code, they tend to encourage premature implementation, produce unnecessary code, and shift costs to maintenance, comprehension, and collaboration.

[### Registry mirror authentication with Kubernetes secrets (4 minute read)](https://www.cncf.io/blog/2026/03/09/registry-mirror-authentication-with-kubernetes-secrets/?utm_source=tldrdevops)

CRI-O credential provider plugins enable Kubernetes to authenticate to private registry mirrors using namespace scoped secrets instead of node level credentials, preserving tenant isolation. The kubelet invokes the plugin during image pulls, which retrieves secrets via service account tokens and generates temporary auth files for CRI-O.
